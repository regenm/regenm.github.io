一、Dante 是什么

Dante（Digital Audio Network Through Ethernet）
是由 Audinate 公司 开发的一种 基于 IP 网络的数字音频传输协议。

一句话总结：

🎧 Dante 就是“音频领域的 NDI” —— 它让音频信号可以通过普通网线在局域网内实时、高保真、低延迟地传输。

Dante 被广泛用于：

  • 专业会议系统
  • 舞台扩声与广播系统
  • 录音棚与演播室
  • 校园、酒店、指挥中心等多音频节点环境

二、Dante 的完整工作流程

下面我们用你熟悉的逻辑结构分析：

音频采集 → 编码 → 网络传输 → 解码 → 播放输出

① 音频采集阶段

  • 麦克风、调音台、音频接口等设备采集到模拟音频信号；
  • 模拟信号经过 ADC（模数转换），变为数字音频流；
  • 支持常见采样率（44.1kHz ~ 192kHz）和位深（16~32bit）。

在传统系统中，这些音频要用一根根模拟线缆连接；
而在 Dante 系统中，这些音频直接走网线。

② 编码阶段

  • 与 H.265 不同，Dante 不进行压缩编码；
  • 它传输的是 线性 PCM 数字音频（无压缩）；
  • 这意味着：
    • 保留了全部音质；
    • 延迟极低（典型 1ms 以下）；
    • 对带宽要求略高（但音频远小于视频）。

💡 例：48kHz、24bit、2声道的 Dante 音频流 ≈ 2.3 Mbps。
所以在千兆局域网内，即使传上百路音频也完全没压力。

③ 传输阶段

  • Dante 使用标准以太网进行传输（无需专用线路），底层走 UDP/IP；
  • 拥有严格的时钟同步机制（IEEE 1588 PTP 精密时间协议），确保所有设备毫秒级同步；
  • 通过 组播或单播 将音频流分发给多台设备；
  • 网络要求：千兆以太网即可，低延迟、无抖动。

🧠 因为是 UDP 传输，Dante 的“稳定性”依靠：

  • 高质量局域网；
  • QoS 优先级控制；
  • PTP 时间同步；
  • 自适应缓冲机制。

④ 解码阶段

  • 接收端设备（如音箱、调音台、电脑）从网络中接收 Dante 流；
  • 通过 DAC（数模转换） 恢复为模拟信号，送入功放或扬声器；
  • 整个过程延迟极低、相位同步。

🎛️ 例如：

  • 一台麦克风 → Dante 网络 → 多个功放/音响；
  • 全部音频同步，无回音、无漂移。

⑤ 控制与管理

  • Dante 配套软件：Dante Controller
    • 自动发现网络内所有 Dante 设备；
    • 像“路由表”一样配置每路音频的输入输出；
    • 实现一对多、多对多音频传输；
    • 支持虚拟声卡（Dante Virtual Soundcard）让普通电脑也能接入 Dante 网络。

三、Dante 的优势

优势点 说明
低延迟 小于 1ms，几乎实时
高音质 无压缩 PCM 传输
基于 IP 网络 使用标准以太网，无需专线
高同步性 使用 PTP 时间同步
灵活路由 一对多、多对多连接
可扩展性强 上百通道音频可共存
软件化控制 用 Dante Controller 管理信号流

四、在视频会议系统中的应用

在现代视频会议系统中，Dante 通常与 NDI / H.265 配合使用：

模块 协议 作用
摄像头视频流 NDI / H.265 高质量视频传输
麦克风音频流 Dante 高保真音频传输
会议控制 IP 网络 统一管理信号路由

📍 举个例子：

  • 会议主讲人的麦克风通过 Dante 网络传到所有会议终端；
  • 摄像头的视频通过 NDI 传给控制台；
  • 控制台再将视频与音频混合，经 H.265 推流至互联网；
  • 所有音视频严格同步，延迟极低。

五、NDI、Dante、H.265 的总体对比

特性 H.265 NDI Dante
类型 视频压缩编码标准 视频网络传输协议 音频网络传输协议
应用范围 公网 / 云传输 局域网视频实时传输 局域网音频实时传输
压缩情况 高压缩（节省带宽） 轻压缩（低延迟） 无压缩（高音质）
延迟 50–200 ms <16 ms <1 ms
传输协议 RTP / RTMP / HLS UDP / TCP UDP / IP
网络需求 中等 高带宽（千兆） 中高（千兆）
典型场景 直播、点播 会议、演播室 会议音频、录音、广播
硬件需求 编解码芯片 CPU/GPU 解码 专用音频接口或网卡

六、一句话总结

🎧 Dante 是专为音频设计的 IP 网络传输协议，
以无压缩、高同步、低延迟为核心，让多路音频能在局域网中自由路由、实时播放。
它是现代视频会议和演播室系统中“声音部分的 NDI”，与 H.265 和 NDI 共同构成完整的音视频链路

1. 查看设备信息

找到设备所在网络，并修改网卡设置。修改好了之后可通过ping命令查看是否在同一网段。如无法ping通，可以尝试换一个静态IP，因为可能发生了IP冲突。
